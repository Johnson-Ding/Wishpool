## 4. 初筛过程

### 4.1 iOS

当前最值得保留的 iOS 方向：

1. `Apple Xcode 26.3 官方 agentic coding / MCP`
2. `XcodeBuildMCP`

判断原因：

- 一条是官方路线
- 一条是当前外部生态里更贴近真实开发链路的 MCP

### 4.2 Android

当前最值得保留的 Android 方向：

1. `Android Studio AI agent + MCP support + Journeys`
2. `gradle-mcp-server`

判断原因：

- Android 官方路线目前更偏 IDE 内置 agent 能力
- 第三方更适合作为构建层补强，而不是完全替代官方 IDE 能力

### 4.3 双端补充

1. `mobile-next/mobile-mcp`
2. `appium/appium-mcp`

判断原因：

- 这两类更偏自动化、设备交互和回归验证
- 适合进入工具组合，但不适合作为“原生开发主基座”
