## 🎯 **Agent 定位**

心愿发布是产品核心链路的专门 Agent，负责从"说出心愿"到"搭子协同"的完整旅程。

**核心使命**: 重构破碎的发愿流程，建立真实可用的发愿→AI方案→搭子匹配→协同履约体系。

## ❗ **当前严重问题**

### 1. 语音转写造假
**问题**: 录音后显示硬编码的"我想去海边放松一下"，不是真实转写
**影响**: 用户无法表达真实心愿，Demo效果欺骗性
**优先级**: P0（立即修复）

### 2. AI方案静态化
**问题**: 方案都是预设数据，没有基于用户输入的动态生成
**影响**: 方案与心愿不匹配，AI体验完全虚假
**优先级**: P0（立即修复）

### 3. 搭子匹配系统缺失
**问题**: US-08/09/10 完全未实现，产品价值断层
**影响**: 失去"一搭子一宇宙"核心体验，用户留存堪忧
**优先级**: P1（重点开发）

### 4. 轮次推进机制空白
**问题**: 48小时推进调度没有真实实现
**影响**: 心愿容易搁置，执行率低下
**优先级**: P1（重点开发）

## 🚀 **重构路线图**

### Phase 1: 修复核心欺骗（1-2天）
**目标**: 消除Demo中的虚假元素，建立真实功能基础

```
✅ 任务1.1: 接入真实语音转写API
  - 替换 MOCK_TRANSCRIPT 为真实 Speech-to-Text
  - 支持录音上传和实时转写
  - 错误处理和重试机制

✅ 任务1.2: 建立AI方案生成后端
  - 创建 ai-plan-generator Edge Function
  - 基于用户心愿动态生成方案
  - 结构化输出（目标、步骤、时间安排）

✅ 任务1.3: 重构场景匹配逻辑
  - 从关键词匹配升级为语义理解
  - 支持开放式心愿输入
  - 智能分类（学习、健康、社交、技能等）
```

### Phase 2: 建立搭子匹配系统（3-5天）
**目标**: 实现产品核心价值"一搭子一宇宙"

```
✅ 任务2.1: 搭子匹配算法
  - 基于心愿类型、位置、时间偏好匹配
  - 用户画像和兴趣标签系统
  - 匹配质量评分和排序

✅ 任务2.2: 匹配界面和交互
  - 搭子推荐卡片（匿名信息 + 标签）
  - 接受/深度调研/拒绝流程
  - 匹配历史和成功率统计

✅ 任务2.3: 协同筹备功能
  - 时间协商界面（日历选择器）
  - 3轮协商限制，避免无限拖延
  - 自动日历提醒和确认机制
```

### Phase 3: 完善执行和反馈（2-3天）
**目标**: 建立完整的履约和反馈循环

```
✅ 任务3.1: 轮次推进系统
  - 48小时自动推进调度
  - 多状态处理（进行中/困难/调整/取消）
  - 个性化建议和鼓励机制

✅ 任务3.2: 履约和反馈
  - 执行提醒和状态追踪
  - 成果上传（图文）
  - 满意度评价和经验分享

✅ 任务3.3: 数据分析和优化
  - 搭子匹配成功率分析
  - 心愿完成率统计
  - 推进频率和用户反馈优化
```

## 📐 **协作边界**

### 可以独立决策的
- 发愿流程的交互逻辑
- 搭子匹配的算法规则
- 推进频率和提醒策略
- 内部组件的设计和实现

### 需要协调的
- **与基础设施 Agent**: 设计组件复用、主题适配
- **与协调者 Agent**: 后端API架构、数据表结构变更
- **与心愿管理 Agent**: 心愿状态更新、数据同步
- **跨端一致性**: Android/iOS 端相同功能的同步开发

### 不能触碰的
- 其他板块的核心文件
- 共享的数据表结构（需协调变更）
- 全局状态管理逻辑
- 其他Agent负责的组件库
