# Android ASR 用户体验优化 - 测试验证方案

## 🎯 优化总结

### 核心改进
1. **智能文本合并**: 新增 `AsrTextManager` 处理 ASR 结果与用户编辑的智能协调
2. **增强状态反馈**: 扩展 `AsrState` 增加置信度和稳定性信息
3. **视觉反馈提升**: 新增 `AsrAwareTextField` 和 `AsrStatusIndicator` 组件
4. **光标位置保持**: 通过 `TextFieldValue` 管理光标选择状态

### 关键优化点
- ✅ 解决了光标位置被 ASR 更新打断的问题
- ✅ 增加了识别过程中的实时状态提示
- ✅ 提供了置信度可视化反馈
- ✅ 实现了用户编辑和 ASR 更新的智能协调

## 🎨 视觉验收标准

### 录音状态指示器
- **录音中**: 红色脉动圆点，边框发光
- **识别中**: 黄色/绿色圆点（根据置信度）
- **处理中**: 蓝色淡出效果
- **完成**: 绿色确认，边框恢复正常

### 文本框状态
- **空闲**: 标准样式
- **ASR活动**: 边框发光，标签文字变化
- **用户编辑**: 保持焦点样式，无干扰

### 状态文字
- **实时**: 显示识别进度和内容预览
- **智能**: 根据置信度显示质量提示
- **友好**: 使用自然语言描述状态

## 📊 性能基准

### 响应时间目标
- 文本更新延迟 < 100ms
- 状态切换动画 < 200ms
- 光标位置恢复 < 50ms

### 资源使用目标
- 内存增长 < 10MB（长时间使用）
- CPU 使用率 < 15%（录音期间）
- 电池消耗与原实现相当

## 🔄 回滚方案

如果优化后出现问题，可以：
1. 将 `PublisherSheet` 恢复使用原有的 `WishpoolTextField`
2. 注释掉 `AsrTextManager` 相关代码
3. 回退 `AsrState` 到原始版本

关键回滚文件：
- `PublisherSheet.kt`: 恢复简单的文本更新逻辑
- `AsrState.kt`: 移除置信度字段
- `SherpaAsrManager.kt`: 恢复原始状态设置

## 📝 已知限制

1. **置信度计算**: 当前基于简单规则，可能需要更精确的算法
2. **多语言支持**: 状态文字暂时只支持中文
3. **无障碍功能**: 需要补充 TalkBack 支持
4. **主题适配**: 颜色值可能需要根据主题调整
