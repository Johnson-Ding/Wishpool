# Wishpool Demo

## 定位

`demo/` 是 Wishpool 的 **mock 数据流程演示场**。

它的职责是：
- 演示产品叙事、页面节奏、状态切换和关键交互
- 用 mock 数据验证主流程是否顺、表达是否成立
- 承担内部评审、方案对齐、视觉和交互验证

它**不再承担**：
- 正式 Web 产品实现
- 真实后端能力收口
- 跨端共享契约的主维护职责
- 发布质量标准下的长期演进

## 边界规则

- `web/` 才是正式 Web 主产品端
- `android/`、`ios/` 是正式原生端
- `supabase/`、`ai-server/` 承担真实数据与 AI 能力
- `demo/` 中允许使用 mock 数据、占位流程和演示态
- 如果某项能力要进入正式交付，应该迁移到 `web/` 或对应正式端，而不是继续堆在 `demo/`

## 使用原则

- 评审流程、讲故事、验证交互：优先用 `demo/`
- 做真实接入、类型契约、生产质量治理：不要以 `demo/` 为主战场
- `demo/` 可以保留少量历史服务端/脚本作为演示资产，但不作为正式后端基线

## 当前维护口径

后续对 `demo/` 的要求收敛为三点：

1. 流程能演示
2. mock 数据稳定
3. 表达清晰，不误导为正式实现
