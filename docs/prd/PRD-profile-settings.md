> **板块版本**：V1
> **最后更新**：2026-03-29
> **总领 PRD**：[PRD-wishpool-v3.md](PRD-wishpool-v3.md)
> **用户故事**：US-14 ~ US-15

---

## 板块定位

设置不是独立主页面，而是从“我的愿望”右上角进入的轻量面板。它承接当前阶段最需要用户感知的应用级能力：会员状态、主题风格入口、log反馈、更新检查。

**核心体验**：轻进入 → 快调整 → 有问题能反馈

## 当前阶段边界

本板块当前**只覆盖**：
- 会员体系（当前默认全员会员）
- 主题风格入口（具体主题定义见 `PRD-theme-switching.md`）
- log反馈
- 更新检查

本板块当前**不覆盖**：
- 实名认证
- 18+ 准入
- 完整账号体系
- 完整隐私与权限模型

## 用户故事

### US-14｜设置面板与会员状态

**作为** 许愿池用户
**我想要** 在“我的愿望”页快速打开设置并看到当前会员状态
**以便于** 明白自己当前可用的应用能力，并进入主题风格等设置项

**业务规则与逻辑**：
1. 设置入口位于“我的愿望”页右上角
2. 点击后打开设置面板，而不是跳转到新的复杂页面
3. 设置面板首屏展示“会员体系”条目与当前状态
4. 当前阶段默认所有用户均处于会员态，不提供购买与续费链路
5. 设置面板同时展示“主题风格”“log反馈”“更新检查”三个设置项

**验收标准**：
- [ ] “我的愿望”页右上角存在设置入口
- [ ] 设置面板展示会员状态与核心权益说明
- [ ] 当前阶段明确展示“默认已开通会员”口径
- [ ] 设置面板能进入主题风格、log反馈、更新检查
- [ ] 不出现购买、续费、取消等尚未开放的链路

---

### US-15｜log反馈与更新检查

**作为** 许愿池用户
**我想要** 在设置里反馈问题并检查当前版本更新
**以便于** 出现问题时能快速求助，并确认自己是否运行在稳定版本上

**业务规则与逻辑**：
1. 设置面板中提供“log反馈”入口
2. 用户点击后可进入轻量反馈入口，提交文字问题描述
3. 系统自动附带必要的基础上下文，例如版本号、平台信息、时间戳
4. 设置面板中提供“更新检查”入口
5. 用户可查看当前版本并手动触发检查更新
6. 若没有新版本，明确告知“当前已是最新版本”；若有新版本，展示更新提示与下一步动作

**验收标准**：
- [ ] 设置面板存在“log反馈”入口
- [ ] 反馈入口允许提交问题描述，并自动附带基础诊断上下文
- [ ] 设置面板存在“更新检查”入口
- [ ] 能展示当前版本号与检查结果
- [ ] 检查更新失败时有明确提示，不让用户停留在不确定状态
