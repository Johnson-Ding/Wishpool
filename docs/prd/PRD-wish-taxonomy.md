# PRD-wish-taxonomy: 愿望分类学

## 文档定位

**类型**：跨板块基础框架文档
**被引用方**：所有板块 PRD（广场、发布、AI Agent、管理、情感对话）
**核心价值**：定义愿望的结构化分类维度，建立产品环路模型，指导各板块的协同设计

---

## 一、两个独立维度

愿望有两个正交的结构属性，决定了系统如何处理它：

### 维度 A — 意图清晰度

用户知不知道自己想要什么。

| 等级 | 定义 | 典型表达 |
|------|------|---------|
| **模糊** | 情绪驱动、方向不明，用户自己也说不清楚 | "周末好无聊"、"感觉生活需要点变化"、"好累啊" |
| **清晰** | 目标明确、可描述，用户知道自己想干什么 | "想学吉他"、"五一去云南"、"想找人打网球" |

### 维度 B — 执行复杂度

实现这件事需要多少步骤和资源。

| 等级 | 定义 | 典型特征 |
|------|------|---------|
| **简单** | 单步或少步骤、低成本、无需多方协调 | 买本书、找个咖啡厅、看场电影 |
| **复杂** | 多步骤、多资源、需协调、有时间线 | 滑雪旅行（订票+装备+教练+天气+同伴）、转行（调研+学习+简历+面试） |

### 维度交叉 → 四象限

| | **简单** | **复杂** |
|---|---|---|
| **模糊** | 灵感区 | 碎碎念区 |
| **清晰** | （原子任务，嵌入体验执行链路） | 体验区 |

> 注：清晰+简单不构成独立的愿望类型——它是体验执行链路中的原子操作（如"订场地"、"买票"、"约教练"），在系统中作为体验的子任务存在。

---

## 二、三个愿望状态

基于四象限，产品定义三个核心愿望状态：

### 🌟 灵感（模糊+简单）

**用户画像**：不知道干嘛，但想做点什么。

**典型场景**：
- "周末不知道干嘛，有什么好玩的？"
- "下雨天好无聊"
- "想出去走走但不知道去哪"

**系统策略**：
- 推荐/灵感激发（广场内容、AI 推荐）
- 低门槛互动（一句话就能发）
- 引导澄清，促进向「体验」转化

**产品角色**：流量入口，负责起量

---

### 🎯 体验（清晰+复杂）

**用户画像**：知道想要什么，但执行链路长、搞不定。

**典型场景**：
- "想去滑雪，但不知道去哪个雪场、怎么订教练"
- "想学吉他，从零开始不知道怎么规划"
- "五一想去云南，预算 2000"

**系统策略**：
- AI 方案生成（拆解为可执行步骤）
- 推进与校验（进度追踪、自动提醒）
- 协同锁定（搭子匹配、资源对接）
- 履约确认（体验完成判定）

**产品角色**：价值交付核心，用户愿意为此付费

---

### 💭 碎碎念（模糊+复杂）

**用户画像**：说不清楚想要什么，但内心有未被满足的复杂需求。

**典型场景**：
- "考研失败了，不知道接下来怎么办"
- "感觉生活需要点变化"
- "好久没有开心过了"

**系统策略**：
- 倾听与共情（AI 陪伴对话）
- 引导发现（通过对话逐步清晰化）
- 温和转化（"要不先给自己放个假？" → 进入灵感或体验）

**产品角色**：情感连接，增强用户粘性

---

## 三、共振 — 环路的连接力

**共振不是一种愿望类型，而是用户之间的连接力。**

共振的本质：看到别人的状态，被触动，产生行动。

### 共振的表现形式

| 共振来源 | 共振行为 | 进入状态 |
|---------|---------|---------|
| 看到别人的体验分享 | "我也想去！" | → 灵感 |
| 看到别人的愿望 | "我帮你实现" | → 体验（帮Ta实现） |
| 看到碎碎念 | "想安慰/回应Ta" | → 碎碎念 |
| 看到求助 | "帮忙买票/办事" | → 直接行动 |

### 共振的产品化

- **心愿广场**的"帮Ta实现"按钮 — 共振→体验
- **漂流瓶/分享** — 体验完成→共振来源
- **评论/加油** — 共振→碎碎念
- **搭子匹配** — 共振→体验（一起去做）

### 共振的核心价值

共振是整个环路的驱动力。没有共振，环路断裂：
- 没有人分享 → 广场没有内容 → 没有灵感来源
- 没有人互助 → 愿望只能自己实现 → 社区属性消失
- 没有人回应 → 碎碎念沉入虚空 → 情感连接断裂

---

## 四、环路模型

### 核心闭环

```
     ┌───────────────── 共振 ──────────────────┐
     │ （看到别人的状态，被触动，产生行动）          │
     │                                          │
     ▼                                          │
  灵感 ──── AI澄清 ────→ 体验                   │
(模糊+简单)              (清晰+复杂)              │
     ▲                      │                   │
     │                   推进+履约                │
   共振                      │                   │
     │                      ▼                   │
  碎碎念 ←── AI陪伴 ──── 完成/分享 ──────────────┘
(模糊+复杂)
```

### 环路运转示例

**主环路（灵感→体验→共振→灵感）**：
1. 小明刷广场，看到有人分享了攀岩体验 → **共振**
2. "好像挺好玩的" → 进入**灵感**态
3. AI 引导："要不要试试？周末有个新手体验课" → 转化为**体验**
4. AI 帮小明找到场馆、预约时间、匹配搭子 → 推进+履约
5. 小明完成攀岩 → 发分享到广场
6. 小红看到小明的分享 → **共振** → 新的循环开始

**辅助环路（碎碎念→灵感→体验）**：
1. 小红深夜发了"好久没开心过了" → **碎碎念**
2. AI 陪伴对话："最近是遇到什么事了吗？" → 倾听
3. 聊了一会，小红说"好像很久没运动了" → 逐步清晰化
4. AI："要不周末去公园走走？" → 进入**灵感**
5. 小红："好，去哪个公园？" → 转化为**体验**

**互助环路（共振→帮Ta实现）**：
1. 小李看到小王发了"想去听音乐节但一个人不敢去" → **共振**
2. 小李点了"帮Ta实现" → 变成小王的搭子
3. AI 帮两人协调时间、买票、规划行程 → **体验**
4. 两人一起完成了音乐节 → 各自分享 → 双倍共振来源

### 环路健康指标

| 指标 | 衡量什么 | 目标 |
|------|---------|------|
| 灵感→体验转化率 | AI 澄清能力 | >15% |
| 体验完成率 | 推进+履约能力 | >40% |
| 完成→分享率 | 共振内容产出 | >30% |
| 分享→共振率 | 内容感染力 | >10% |
| 碎碎念→灵感转化率 | AI 陪伴引导能力 | >20% |
| 共振→行动率 | "帮Ta实现"等互动 | >5% |

## 五、与现有板块的映射

### 板块对应关系

| 板块 | 主要服务 | 环路中的角色 |
|------|---------|------------|
| 心愿广场 | 灵感 + 共振 | 内容消费与灵感激发，共振的主要发生场所 |
| 发愿流程 | 灵感 → 体验转化 | AI 引导澄清，结构化意图，生成方案 |
| AI Agent 执行 | 体验 | 拆解任务、推进校验、自动/委托执行 |
| 心愿管理 | 体验 | 进度跟踪、状态管理、履约确认 |
| 情感对话 | 碎碎念 | 倾听共情、引导发现、逐步清晰化 |
| 分享/漂流瓶 | 共振 | 体验完成的内容沉淀，触发下一个循环 |

### 对 AI Agent（US-23~29）的影响

US-28 意图分类系统需要**双标签体系**：

- **内容领域标签**（保留现有分类）：情感倾诉、旅行、本地生活、个人成长、执行
- **状态标签**（新增）：灵感 / 体验 / 碎碎念

两套标签独立正交：
- "想去云南" = 旅行 × 体验
- "好无聊想出去" = 本地生活 × 灵感
- "考研失败了" = 个人成长 × 碎碎念

**状态标签决定系统处理策略，内容领域标签决定推荐和知识库匹配。**

## 六、产品架构 — 环路引擎 + 垂类插拔

### 核心理念

许愿池不是一个"周末运动 APP"或"旅行 APP"，而是一个**环路引擎**。

```
┌─────────────────────────────────────────┐
│         许愿池 Wishpool（统一产品）        │
├─────────────────────────────────────────┤
│  环路引擎（产品骨架）                      │
│  灵感 → 体验 → 共振 → 回流               │
│  + AI 引擎（澄清/推进/陪伴）              │
├─────────────────────────────────────────┤
│  垂类内容层（可插拔）                      │
│  ├── 新手友好运动：攀岩体验、飞盘、瑜伽    │
│  ├── 技能学习：烘焙、摄影、手冲咖啡        │
│  ├── 垂类 N: ...（按数据表现扩展）        │
│  └── 垂类之间可交叉（学摄影的拍攀岩的）    │
└─────────────────────────────────────────┘
```

**产品和垂类是分离的：**
- **产品层**：四个状态 + 共振连接 + AI 能力 — 不随垂类变化
- **内容层**：填什么垂类进去 — 可以测试多个，看哪个先转起来
- **垂类选错不致命**：换一个垂类重新灌内容，环路引擎不用改

### 首批垂类方向

**新手友好运动**：攀岩体验、飞盘入门、瑜伽工作坊
- 执行链路标准化（场地+教练+时间=就能成）
- 天然社交属性（容易产生共振）
- 完成感强（拍照分享意愿高）

**技能学习**：周末学烘焙、学摄影、学手冲咖啡
- 有成果物（做出来的东西就是分享素材）
- 复购率高（学了一次想学下一个）
- 碎碎念入口自然（"好羡慕会做饭的人" → 灵感 → 体验）

**垂类间协同**：学了摄影的人去拍攀岩的人 → 跨垂类共振

### 分阶段落地

**Phase 1 — 环路引擎 + 首个垂类验证**

目标：跑通完整环路（灵感 → 体验 → 分享 → 共振 → 灵感），验证环能否自转。

- 环路引擎：四个状态的完整交互闭环
- AI 引擎：澄清 + 推进 + 陪伴的基础能力
- 首个垂类：选一个灌入内容，验证转化
- 碎碎念：基础 AI 陪伴对话能力（Day 1 就做）
- 验证指标：有人完成体验并分享 → 是否真的带来新用户进入灵感态

**Phase 2 — 多垂类扩展 + 深化共振**

- 根据 Phase 1 数据，选择表现最好的垂类优先扩展
- 更丰富的共振形式（搭子匹配、互助、漂流瓶）
- 更深的碎碎念能力（情感分析、长期陪伴）

**Phase 3 — 环路自动化与规模化**

- AI 自动识别愿望状态并路由到最优处理策略
- 环路指标自动监控和优化
- 供给侧规模化（场地、教练、服务对接）
