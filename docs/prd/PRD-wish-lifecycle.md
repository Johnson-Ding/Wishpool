# PRD — 愿望生命周期（V4）

> **版本**：V4.0  
> **状态**：当前基准

---

## 目标

V4 的愿望不再跳到独立策划页，而是在**对话流中完成全生命周期**：
- 发愿 → 生成卡片
- 澄清 → 更新卡片
- 推进 → 勾选 todo
- 完成 → 标记状态

---

## 发愿入口

### 三种发愿方式

| 方式 | 入口 | 结果 |
|---|---|---|
| 气泡选择 | 群聊页单击许愿条 | 弹出 4-6 个轻量方向，点击后生成愿望卡片 |
| 语音输入 | 群聊页长按许愿条 / 首页长按荧光圈 | 语音转文字后生成愿望卡片或碎碎念卡片 |
| 文字输入 | 群聊页输入框 | 直接在对话流中输入，AI 判断后生成卡片 |

---

## 愿望卡片结构

愿望卡片在对话流中展示，包含：
- **标题**：一句话概括愿望
- **AI 总结**：一句话说明这个愿望的意义
- **todo 列表**：2~3 个最小可执行动作
- **进度文案**：如"0/3 已完成"

### 卡片交互
- 点击卡片 → 展开 todo 列表
- 勾选 todo → 实时更新进度
- 点击"去处理" → 回到对应群聊场景

---

## 碎碎念卡片结构

碎碎念卡片在对话流中展示，包含：
- **标题 / 标签**：如"今天的小确幸"
- **正文**：用户说的话
- **状态**：可编辑 / 已分享

### 卡片交互
- 点击卡片 → 展开完整正文
- 点击"分享到广场" → 标记为已分享

---

## 我的愿望 Tab

### 分组展示
- **待开始**：还没开始推进的愿望
- **进行中**：至少勾选了一个 todo 的愿望
- **已完成**：所有 todo 都勾选的愿望

### 交互规则
- 点击愿望 → 展开 todo 列表
- 勾选 todo → 实时更新进度和状态
- 点击"去处理" → 回到对应群聊场景

---

## 当前 demo 实现原则

- 所有愿望和 todo 均为 mock 数据
- 不要求接真实 AI 生成
- 重点验证：卡片生成逻辑、todo 勾选、状态流转
