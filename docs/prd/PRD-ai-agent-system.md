## 板块定位

AI 智能助手系统是 Wishpool 的**大脑**，负责从用户的语音输入到愿望落地执行的全程智能化服务。它不仅要准确理解用户意图，更要在执行过程中提供贴心的判断、适时的汇报和安全的自动执行能力。

**核心体验**：听懂你想要什么 → 智能规划执行路径 → 安全自动执行 → 贴心进度汇报

**产品定位**：从"AI 生成方案"升级为"AI 全程陪伴执行"——用户只需表达愿望，剩下的交给 AI 智能助手完成。

**价值主张**：
- **理解更准确**：不再是关键词匹配，而是深度理解用户真实意图
- **执行更自动**：AI 可直接完成购买、预约、下单等实际操作
- **委托更智能**：做不了的事情，AI 懂得何时求助亲友、何时发布到社区
- **汇报更贴心**：知道什么时候该告诉你进展，什么时候该安静执行

## 用户故事

### US-23｜智能意图理解与确认

**作为** 许愿池用户
**我想要** AI 准确理解我的语音表达并确认意图
**以便于** 避免误解，确保 AI 按我的真实需求执行

**场景描述**：
用户说出心愿后，AI 不再简单匹配关键词，而是深度理解用户的真实意图，识别目标、约束、偏好和隐私敏感度，并向用户确认理解是否正确。

**交互流程**：
1. 用户语音输入心愿（"我想学吉他"）
2. AI 分析并结构化理解：
   - 目标：学会弹奏吉他基础曲目
   - 约束：预算有限，时间主要在周末
   - 偏好：更倾向线下学习
   - 隐私级别：可公开分享学习过程
3. AI 向用户确认："我理解您想要学吉他，预算不高，周末有时间，希望线下学习。这样理解对吗？"
4. 用户确认或纠正，AI 基于反馈调整理解
5. 确认无误后进入执行规划阶段

**验收标准**：
- [ ] AI 能从自然语言中提取目标、约束、偏好、隐私级别
- [ ] 理解结果以自然语言向用户确认
- [ ] 用户可纠正误解，AI 能基于反馈调整
- [ ] 不确定时主动询问，不做假设
- [ ] 涉及敏感信息时询问隐私偏好

### US-24｜智能执行路径判断

**作为** 许愿池用户
**我想要** AI 自动判断最优的执行方式
**以便于** 不用我操心具体怎么做，AI 选择最合适的执行路径

**场景描述**：
AI 确认用户意图后，自动分析任务特性，判断采用 L1（AI自动执行）、L2（亲友代执行）或 L3（网友帮忙）哪种路径最合适。

**执行分级逻辑**：
- **L1 AI自动执行**：标准化任务（购买、预约、查询）
- **L2 亲友代执行**：本地化任务、需要信任关系的任务
- **L3 网友帮忙**：需要人力协助但可脱敏的任务

**交互流程**：
1. 用户确认意图后，AI 分析任务可行性
2. AI 展示执行计划："我可以帮您：购买入门教材（¥200）、预约试听课程、推荐附近的吉他老师"
3. 明确告知执行方式："购买和预约我可以直接完成，老师推荐我会请教有经验的朋友"
4. 用户确认后，AI 开始分层执行
5. 过程中遇到无法自动完成的环节，智能切换委托路径

**验收标准**：
- [ ] AI 能准确判断任务的复杂度和可自动化程度
- [ ] 提前告知用户采用哪种执行方式及原因
- [ ] 执行中能动态切换路径（如自动失败后切换到委托）
- [ ] 委托前询问用户偏好（亲友 vs 网友）
- [ ] 网友委托时自动脱敏处理

### US-25｜智能汇报时机控制

**作为** 许愿池用户
**我想要** AI 在合适的时候告诉我进展，不要频繁打扰
**以便于** 及时了解重要进展，但不被无关信息干扰

**场景描述**：
AI 在执行任务过程中，基于进展重要性、用户状态、紧急程度等因素，智能判断何时汇报、何时静默执行。

**汇报策略**：
- **立即汇报**：关键节点完成、遇到问题需用户决策、金钱支出确认
- **定时汇报**：每日晚间总结进展（如有变化）
- **静默执行**：常规信息收集、背景任务进行、预期内的等待

**交互流程**：
1. AI 开始执行用户确认的计划
2. 遇到不同类型事件时，智能判断汇报时机：
   - 成功购买教材 → 立即汇报（涉及支出）
   - 正在搜索吉他老师 → 静默执行
   - 找到3个候选老师 → 定时汇报（非紧急选择）
   - 遇到缺货需换品牌 → 立即汇报（需用户决策）
3. 汇报消息简洁明了，包含关键信息和下一步计划
4. 用户可随时主动询问详细进展

**验收标准**：
- [ ] 涉及金钱支出时必须立即汇报并确认
- [ ] 需要用户决策时及时汇报，提供选项
- [ ] 正常进展中的例行步骤静默执行
- [ ] 每日晚间汇总当日主要进展（如有）
- [ ] 用户可主动查询详细执行日志

### US-26｜AI 自动执行能力

**作为** 许愿池用户
**我想要** AI 能直接完成购买、预约等实际操作
**以便于** 真正解放双手，不只是给建议而是实际帮我做事

**场景描述**：
AI 具备调用外部 API 的能力，可以完成电商购买、在线预约、信息查询等标准化操作，让用户体验到真正的"全自动执行"。

**自动执行能力边界**：
- **可以做**：网购下单、预约服务、查询信息、发送消息
- **不能做**：涉及身份验证的操作、不可逆的重要决策、超出预算的支出
- **需要确认**：首次使用新服务、超过设定金额、涉及个人隐私

**交互流程**：
1. AI 识别可自动执行的任务环节
2. 首次操作时询问授权："我需要访问您的购物账户来完成下单，是否授权？"
3. 获得授权后，AI 自动执行：查找商品 → 比较价格 → 选择优质商家 → 下单购买
4. 执行过程实时记录，重要节点立即汇报
5. 完成后提供执行摘要和后续安排

**验收标准**：
- [ ] 支持主流电商平台自动购买
- [ ] 支持在线服务预约（美团、大众点评等）
- [ ] 首次使用新平台时征得用户授权
- [ ] 自动控制单次支出上限（可用户设置）
- [ ] 提供详细的执行记录和凭证

### US-27｜智能委托与脱敏处理

**作为** 许愿池用户
**我想要** AI 在无法自动完成任务时，智能选择合适的帮助渠道
**以便于** 即使 AI 做不了，也能通过其他方式完成愿望

**场景描述**：
当 AI 遇到无法自动执行的任务时，智能判断寻求亲友帮助还是发布到社区求助，并自动处理隐私脱敏。

**委托策略**：
- **亲友优先**：本地化任务、需要信任的任务、隐私敏感任务
- **社区求助**：技能分享、资源互换、经验咨询类任务
- **脱敏原则**：删除姓名、地址等个人信息，保留任务核心需求

**交互流程**：
1. AI 判断任务无法自动完成
2. 分析任务特性，选择委托渠道：
   - "需要推荐本地靠谱吉他老师" → 建议亲友帮助
   - "寻找吉他学习经验分享" → 发布到心愿广场
3. 脱敏处理：移除个人信息，保留地区和需求描述
4. 向用户展示脱敏后的求助内容
5. 用户确认后发布，AI 持续跟进回复

**验收标准**：
- [ ] 智能判断任务适合亲友帮助还是社区求助
- [ ] 自动脱敏处理，移除个人隐私信息
- [ ] 脱敏内容展示给用户确认后再发布
- [ ] 发布后跟进社区回复，筛选有效建议
- [ ] 亲友委托时保留完整信息但征得同意

### US-28｜AI 质量基准与持续优化

**作为** 许愿池用户
**我想要** AI 的理解和回答质量持续提升
**以便于** 获得越来越好的服务体验

**场景描述**：
建立 AI 服务质量的评估体系，通过用户反馈和系统监控，持续优化 AI 的理解准确性、回答质量、安全边界遵守和个性化水平。

**质量评估四个维度**：

**1. 意图分类准确性**
- 情感倾诉类：倾诉压力、吐槽烦恼、寻求安慰 → 识别为情绪支持需求，以倾听和引导为主
- 旅行类：出行规划、订票订酒店、行程安排 → 识别为旅行需求，全链路执行
- 本地生活类：周末去哪、找餐厅、休闲活动 → 识别为本地消费/体验需求，推荐+预约
- 个人成长类：学技能、考证、职业发展、养成习惯 → 识别为成长需求，方案+轮次推进
- 执行类：买东西、办手续、查信息、预约服务 → 识别为线上事务需求，AI 直接执行

**2. 回答清晰性和准确性**
- 必须包含：目标、时间线、预算估算、风险提示
- 语言简洁：核心信息在前30秒内传达完毕
- 结构清晰：分步骤、有优先级、可操作
- 准确性：信息来源可靠，不编造数据

**3. 安全边界遵守**
- 金钱操作：单次¥500以上必须确认，累计¥2000以上必须重新授权
- 隐私保护：个人信息、位置、联系方式严格脱敏
- 风险提示：涉及健康、法律、投资必须提醒专业咨询
- 伦理约束：不执行违法、有害、欺骗性任务

**4. 个性化调用用户 context**
- 学习偏好：根据用户历史愿望推断学习风格（在线vs线下、深度vs广度）
- 时间约束：识别用户的空闲时间模式（工作日晚上、周末全天）
- 预算范围：基于历史消费推断可接受的支出水平
- 社交偏好：判断用户倾向独立完成还是寻求搭子协作

**验收标准**：
- [ ] 意图分类准确率 >90%（100个测试样本）
- [ ] 回答必备要素完整率 >95%（目标+时间+预算+风险）
- [ ] 安全边界零违反（金钱、隐私、风险提示）
- [ ] 个性化程度提升（新用户vs回头用户的方案差异度 >50%）
- [ ] 用户满意度评分 >4.2/5.0

### 回答质量示例库（按意图分类）

#### 情感倾诉类回答示例

**场景：大学生说"考研失败了，不知道接下来怎么办"**

**✅ 好回答**：
"考研没上岸，确实挺难受的。准备了那么久，这种失落感很正常。
你现在想先聊聊心情，还是想一起看看接下来有哪些选择？不着急，按你的节奏来。"

**❌ 坏回答**：
"没关系，你可以二战或者找工作。"（跳过情绪，直接给方案）
"考研失败的人很多，不用太在意。"（否定情绪）

**核心原则**：情感倾诉类的第一响应永远是**共情**，不是方案。

#### 旅行类回答示例

**场景：大学生说"五一想跟室友去云南玩，预算2000"**

**✅ 好回答**：
"云南五一很热门！帮你们规划一下：
- 机票/高铁：我先查一下最优价格
- 住宿：推荐大理/丽江的青旅，人均50-80/晚
- 行程：大理2天+丽江2天比较合理
- 预算分配建议：交通800+住宿400+吃饭400+景点门票400

我先帮你们查5.1-5.5的往返票价，查到后告诉你。"

**❌ 坏回答**：
"云南很美，推荐去大理和丽江。"（没有执行，只是建议）

#### 本地生活类回答示例

**场景：白领说"周六想带孩子去个有意思的地方"**

**✅ 好回答**：
"帮你看了一下周六的情况：
- 天气：晴，25°C，适合户外
- 推荐3个选项：
  1. XX科技馆（亲子互动区，适合3-8岁）

  3. XX公园儿童乐园（免费，新开的）

你家孩子多大？我帮你筛选最合适的，选好后直接帮你订票。"

**❌ 坏回答**：
"可以去公园、博物馆、游乐场。"（泛泛推荐，无本地化信息）

1. 你有常去的口腔医院吗？还是帮你就近推荐？

确认后我直接帮你挂号。"

**❌ 坏回答**：

#### 回答质量评估标准

| 质量维度 | 评分标准（1-5分） | 各类别重点 |
|---------|-----------------|-----------|
| **意图识别** | 正确识别主分类+情绪浓度 | 情感倾诉类：不能跳过情绪直接给方案 |
| **响应匹配** | 回答方式匹配意图类别 | 执行类：不能只给建议不执行 |
| **信息完整** | 包含关键决策要素 | 旅行类：预算+时间+行程缺一不可 |
| **执行力** | 是否推进到下一步行动 | 本地生活类：推荐后要能预约 |
| **边界感** | 不越界、不过度 | 情感倾诉类：不评判、不说教 |

#### 常见回答错误类型

| 错误类型 | 错误示例 | 正确做法 |
|---------|----------|---------|
| **情绪跳过** | 用户倾诉时直接给方案 | 先共情再探索需求 |
| **只建议不执行** | "你可以去XX平台预约" | 直接帮用户预约 |
| **泛泛推荐** | "推荐去公园、博物馆" | 基于位置、天气、人群给具体推荐 |
| **信息缺失** | 旅行方案没有预算分配 | 必须包含预算、时间、行程要素 |
| **越界干预** | 在情感倾诉中说"你应该分手" | 引导思考，不替用户做决定 |
| **忽略上下文** | 给大学生推荐人均500的餐厅 | 调用用户 context 做个性化适配 |

### US-29｜执行分级体系与用户体验

**作为** 许愿池用户
**我想要** 清楚了解 AI 会如何完成我的愿望
**以便于** 对执行过程有合理预期，感到安心和可控

**场景描述**：
建立清晰的执行分级体系，让用户明白什么任务 AI 能直接完成，什么需要寻求帮助，以及如何保护隐私。

**执行分级详细设计**：

**L1 - AI 自动执行**
- **适用场景**：标准化、可编程、低风险的任务
- **典型任务**：网购下单、在线预约、信息查询、发送通知
- **用户感知**："我正在帮您..."（进度条 + 实时状态）
- **授权机制**：首次使用需授权，后续在预设范围内自动执行
- **安全边界**：单次¥500、总计¥2000、无身份敏感操作

**L2 - 亲友代执行**
- **适用场景**：本地化、需要信任关系、隐私敏感的任务
- **典型任务**：推荐本地商家、代为询问、线下协助
- **用户感知**："我会请您的朋友帮忙..."（显示委托对象）
- **脱敏策略**：保留完整信息，但征得双方同意
- **反馈机制**：朋友响应后及时通知用户结果

**L3 - 网友帮忙**
- **适用场景**：经验分享、资源互换、众包任务
- **典型任务**：经验咨询、资源推荐、技能指导
- **用户感知**："我会在社区发布求助..."（显示脱敏后内容）
- **脱敏策略**：严格删除个人信息，保留地区和需求核心
- **激励机制**：帮助他人可获得社区声望值

**分级判断逻辑**：
```
IF 标准化API可处理 → L1
ELIF 需要本地知识且用户有指定亲友 → L2
ELIF 可脱敏且适合社区众包 → L3
ELSE 告知用户暂时无法完成，建议手动处理
```

**用户体验设计**：

**透明度设计**：
- 执行前预告："我将通过XX方式帮您完成"
- 执行中状态："正在XX平台为您下单..."
- 委托时说明："因为需要本地经验，我请朋友帮您询问"

**控制感设计**：
- 任何时候都可暂停："暂停执行"按钮始终可见
- 委托前确认："以下内容将发送给您的朋友，是否同意？"
- 结果可追溯：完整的执行记录和决策依据

**验收标准**：
- [ ] L1自动执行成功率 >85%（标准化任务）
- [ ] L2亲友响应时效 <4小时（工作时间）
- [ ] L3社区回复率 >60%（发布后24小时内）
- [ ] 分级判断准确率 >90%（AI选择合适的执行层级）
- [ ] 用户对执行方式的理解度 >4.0/5.0（事后调研）
- [ ] 脱敏质量：0个人信息泄露事件

## 产品边界与约束

### 能力边界清单

**✅ AI 可以做的事**
- 标准化购买（图书、数码产品、日用品）
- 在线服务预约（餐厅、美容、培训）
- 信息查询与整理（价格比较、评价汇总）
- 学习计划制定与进度跟踪
- 社区求助发布与回复筛选

**❌ AI 不能做的事**
- 涉及身份验证的操作（银行开户、政务办事）
- 医疗诊断与治疗建议
- 法律咨询与投资理财建议
- 涉及他人隐私的操作
- 违法、有害、欺骗性的任务

**⚠️ AI 需要确认才能做的事**
- 首次使用新的外部服务
- 单次支出 >¥500 或累计 >¥2000
- 涉及个人隐私信息的委托
- 不可逆的重要操作
- 超出预设执行范围的任务

### 伦理约束原则

**1. 用户自主权优先**
- 用户随时可以停止、修改、撤销 AI 的执行
- 重要决策必须征得用户明确同意
- 不得替用户做出价值观判断

**2. 隐私保护至上**
- 个人信息严格脱敏处理
- 委托给第三方时最小化信息披露
- 用户有权知晓信息的使用范围

**3. 安全第一**
- 涉及金钱、健康、法律的操作必须谨慎
- 提供风险提示和专业建议渠道
- 不执行可能造成伤害的任务

**4. 诚实透明**
- 如实告知 AI 的能力限制
- 执行失败时说明真实原因
- 不编造信息或过度承诺

### 用户同意机制

**分层同意模型**：

**一次性授权（注册时）**
- AI 基础执行能力说明
- 数据使用与隐私政策
- 执行边界和安全机制

**单次确认（每次执行前）**
- 具体执行计划预览
- 涉及的平台和权限
- 预算和风险提示

**持续同意（执行过程中）**
- 超出预期的额外操作
- 遇到问题需要调整方案
- 委托第三方时的信息授权
