# 消息推送 — 板块 PRD

> **板块版本**：V1
> **最后更新**：2026-03-29
> **总领 PRD**：[PRD-wishpool-v3.md](PRD-wishpool-v3.md)
> **用户故事**：US-16 ~ US-19

---

## 板块定位

消息推送是产品的"主动触达"能力——用户不打开 App 时，系统仍然在推进心愿。48h 轮次提醒、搭子匹配通知、社区互动反馈，都依赖这个板块。没有推送，心愿容易被遗忘。

**核心体验**：及时触达 → 回到产品 → 继续推进

---

## 用户故事

### US-16｜轮次推进提醒

**作为** 许愿池用户
**我想要** 在每个48h轮次到来时收到提醒
**以便于** 及时更新进展，保持执行节奏

**场景描述**：
轮次推进是心愿执行的核心节奏。每48小时，系统需要提醒用户更新进展。提醒应该有温度，不是冷冰冰的系统通知，而是像搭子在关心你。

**推送内容**：
- 首次轮次："你的心愿「XXX」方案已就绪，准备好开始了吗？"
- 常规轮次："「XXX」进展如何？来聊聊吧 🌙"
- 停滞提醒："「XXX」已经X天没更新了，需要调整计划吗？"

**交互流程**：
1. 系统在48h节点触发推送
2. 用户点击推送进入对应愿望的轮次更新页
3. 快速选择状态或输入进展
4. 系统设置下一个轮次

**验收标准**：
- [ ] 48h 周期自动触发推送通知
- [ ] 推送文案有温度，包含愿望标题
- [ ] 点击推送直接进入对应愿望页面
- [ ] 连续无响应时降低推送频率（避免骚扰）
- [ ] 用户可自定义推送时间偏好（如只在晚间推送）

---

### US-17｜搭子匹配通知

**作为** 许愿池用户
**我想要** 在有搭子匹配结果时收到通知
**以便于** 及时查看和决策，不错过合适的搭子

**推送场景**：
- 系统匹配到候选搭子："为你的心愿「XXX」找到了可能的搭子，来看看？"
- 搭子接受了你的邀请："有人愿意和你一起完成「XXX」！"
- 搭子发起时间协商："你的搭子提议了一个时间，看看合不合适？"
- 协商达成一致："时间确定了！准备一起出发吧 ✨"

**验收标准**：
- [ ] 匹配结果即时推送
- [ ] 协商流程中的每个节点有对应推送
- [ ] 点击推送直接进入匹配/协商页面
- [ ] 推送包含足够上下文，不需要打开 App 就知道大概情况

---

### US-18｜社区互动通知

**作为** 许愿池用户
**我想要** 在有人与我互动时收到通知
**以便于** 参与社区对话，增强归属感

**推送场景**：
- 有人给我的内容点赞
- 有人评论了我的内容
- 有人回复了我的漂流瓶
- 我参与的"帮Ta实现"任务有了结果

**验收标准**：
- [ ] 互动类通知支持合并（"X人赞了你的心愿"而非逐条推送）
- [ ] 点击进入对应内容详情
- [ ] 用户可选择关闭社区通知

---

### US-19｜系统消息管理

**作为** 许愿池用户
**我想要** 统一管理所有收到的消息和通知
**以便于** 不遗漏重要信息，按需处理

**场景描述**：
App 内需要一个消息中心，汇总所有推送历史。用户可以在这里查看未读消息、历史通知，管理通知偏好。

**消息分类**：
- **心愿动态**：轮次提醒、状态变更
- **搭子消息**：匹配、协商、协同
- **社区互动**：点赞、评论、回复
- **系统公告**：版本更新、活动通知

**验收标准**：
- [ ] 提供统一的消息中心入口
- [ ] 消息按类型分 Tab 展示
- [ ] 未读消息有角标提示
- [ ] 支持一键已读
- [ ] 提供通知偏好设置（按类型开关）
