## 板块定位

"我的愿望"是用户的个人主阵地——所有发过的愿望、当前状态、下一步动作，以及应用级设置入口，都从这里被承接。它不是一个被动的列表页，而是用户推进心愿、回看历史、进入设置的稳定容器。

**核心体验**：一目了然 → 快速决策 → 持续推进

## 页面结构

"我的愿望"页当前收敛为四个稳定区域：

1. **顶部身份区**：展示当前陪伴角色头像与页面标题
2. **右上角设置入口**：进入设置面板，承接会员状态、主题风格、log反馈、更新检查
3. **愿望分组列表**：待决策 / 进行中 / 已完成
4. **历史与回看延展位**：为后续回顾、故事沉淀、成长记录预留

> 设置入口是“我的愿望”页的辅助入口，不替代愿望管理本身。

## 用户故事

### US-11｜愿望列表、状态管理与设置入口

**作为** 许愿池用户
**我想要** 查看我所有愿望的状态和进展
**以便于** 了解当前需要处理的事项，管理我的愿望

**场景描述**：
"我的愿望" Tab 展示用户的所有愿望，按状态分组显示；用户既可以快速了解哪些需要决策、哪些正在进行、哪些已经完成，也可以从右上角进入设置面板管理应用层能力。

**列表分组**：
- **待决策**：需要用户确认方案、选择搭子等（对应状态：`draft` / `clarifying` / `planning` / `partner_matching`）
- **进行中**：正在执行的愿望（对应状态：`collaborating` / `ready` / `in_progress`）
- **已完成**：完成或放弃的愿望（对应状态：`completed` / `failed`）

**交互流程**：
1. 用户点击"我的愿望" Tab
2. 看到顶部身份区、右上角设置入口和按状态分组的愿望列表
3. 每个愿望卡片显示基本信息和状态
4. 点击卡片进入详情页
5. 点击右上角设置入口，打开设置面板

**验收标准**：
- [ ] 按状态分组显示愿望列表
- [ ] 每个分组显示愿望数量
- [ ] 卡片显示愿望标题、状态标签、最近更新时间
- [ ] 右上角存在清晰的设置入口
- [ ] 点击设置后在当前上下文内打开设置面板，不打断愿望主流程
- [ ] 支持下拉刷新获取最新状态
- [ ] 空状态有友好的引导文案（"还没有心愿？试试发一个吧"）

### US-12｜愿望详情与进度跟踪

**作为** 许愿池用户
**我想要** 查看单个愿望的详细信息和执行进度
**以便于** 了解完整情况，做出下一步决策

**详情页内容**：
- 原始愿望描述
- AI 生成的方案
- **待决策卡片区**：搭子提议换时间、资源变动方案、异常处理选择等
- **进展时间线**：每次48h check-in自动沉淀的记录，可查看完整执行历程
- **搭子动态**：搭子完成了第X步、发来消息、状态变更等实时更新
- **异常处理**：搭子退出→AI补位、资源变动→重规划、目标调整→回到澄清
- **记录导出**：全链路沉淀的记录支持一键导出（非独立功能）

**交互功能**：
- 待决策事项的快速处理（接受/拒绝/需要更多信息）
- 进展状态更新（继续/调整/暂停/遇到问题）
- 异常情况的引导处理
- 与搭子的协同沟通
- 历史记录的导出

**验收标准**：
- [ ] 待决策卡片区突出显示需要用户处理的事项
- [ ] 进展时间线自动记录每次check-in内容，按时间正序排列
- [ ] 搭子动态实时更新，区分"系统通知"和"搭子消息"
- [ ] 异常处理提供明确的选择项和后续流程指引
- [ ] 记录导出功能入口明显，支持多种格式（文本/图片）
- [ ] 页面信息分层清晰，重要信息优先显示
- [ ] 所有操作都有明确的反馈和确认机制

### US-13｜历史归档与回顾

**作为** 许愿池用户
**我想要** 回顾我过去完成的愿望
**以便于** 获得成就感，总结经验，规划未来

**功能特性**：
- 完成愿望的成果展示
- 执行时间统计
- 协同搭子历史
- 个人成长记录
- 愿望类型分析

**验收标准**：
- [ ] 提供历史愿望的检索和筛选
- [ ] 展示个人完成统计数据
- [ ] 支持回顾和分享历史成果
- [ ] 生成个人成长报告（月度/年度）
