> **版本**：V4.0  
> **最后更新**：2026-04-06  
> **状态**：当前重构基准  
> **替代**：PRD-wishpool-v3.md 的主流程定义

---

## 产品定位

**许愿池 Wishpool** —— 一个把“模糊愿望 / 随口碎碎念”收进对话流、再慢慢推成行动的陪伴式产品。

V4 的核心不再是“发愿后跳到独立策划页”，而是：
- 用户进入默认三角色群聊
- 在聊天里自然说出愿望或碎碎念
- AI 在对话流里直接生成卡片
- 用户在卡片上继续推进、修改、完成

**核心价值**：把愿望从“说出来”到“开始做”的距离，缩到同一个对话流里。

## 底部导航结构

底部导航固定为：**广场｜许愿｜我的**

### 许愿入口形态

| 所在页 | 中间入口形态 | 说明 |
|---|---|---|
| 广场 / 我的 | 荧光圈（圆形） | 视觉重心，吸引进入群聊 |
| 群聊详情 | 许愿荧光条（横条） | 荧光圈向下吸附后变形，常驻底栏 |

### 许愿入口交互

- **首页单击荧光圈** → 进入默认群聊详情页
- **首页长按荧光圈** → 进入群聊详情页，并直接唤起语音输入
- **群聊页荧光条存在三态**：流动｜许愿｜碎碎念
- **群聊页 A 态“流动”单击荧光条** → 打开双列气泡（左列愿望，右列碎碎念）
- **群聊页 B 态“许愿”单击荧光条** → 直接生成愿望卡片
- **群聊页 C 态“碎碎念”单击荧光条** → 直接生成碎碎念卡片
- **群聊页长按荧光条** → 打开语音输入
- **B / C 态仅由 AI 明确引导触发，点击完成后立即恢复 A 态“流动”

## 核心屏幕流

```text
Screen 1: Splash（启动页）
  → 用户点击任意位置 / 自动进入
  → Screen 2

Screen 2: 广场 Tab（灵感激发场）
  - 展示心愿卡片 + 碎碎念卡片（穿插，mock 数据）
  - 底部：荧光圈（圆形）
  → 单击荧光圈 → Screen 3
  → 长按荧光圈 → Screen 6（经由 Screen 3 承接）

Screen 3: 群聊详情页（默认三角色群聊）
  - 顶部：三个角色头像（横排，紧密长条）
  - 中间：对话流
    - 用户消息
    - AI 消息（角色头像 + 昵称）
    - 愿望卡片
    - 碎碎念卡片
  - 底部：输入框一行 + 底部导航一行
  - 底栏中间：许愿荧光条
  → 单击头像 → Screen 4
  → 单击许愿条 → Screen 5
  → 长按许愿条 → Screen 6

Screen 4: 角色卡片（浮层）
  - 显示角色详情
  - 左右滑动循环切换：眠眠月 / 朵朵云 / 芽芽星
  → 点击外部关闭 → 回到 Screen 3

Screen 5: 发愿气泡（浮层）
  - 仅在荧光条处于 A 态“流动”时打开
  - 左列：愿望方向；右列：碎碎念方向
  → 点击左列项 → 回到 Screen 3，并在对话流中生成愿望卡片
  → 点击右列项 → 回到 Screen 3，并在对话流中生成碎碎念卡片
  → 点击外部关闭 → 回到 Screen 3

Screen 6: 语音输入（底部半圆蒙层）
  - 波形动画
  - 实时 / mock 转写文案
  → 完成输入 → 回到 Screen 3，由 AI 回应并引导用户点击下方荧光条生成对应卡片
  → 取消 → 回到 Screen 3

Screen 7: 我的愿望 Tab
  - 分组显示：待开始 / 进行中 / 已完成
  - 愿望展开后显示 todo
  - 勾选 todo 后实时更新进度
```

## 信息结构变化

### V3 → V4 的核心变化

| V3 | V4 |
|---|---|
| 发愿后进入独立策划页 | 发愿后直接在对话流里生成卡片 |
| 需要澄清页 / 半屏聊天澄清 | 默认群聊就是主容器 |
| 愿望、方案、推进分散在多屏 | 愿望卡片在对话流中持续存在 |
| 对话列表是潜在入口 | 只有一个默认群聊，不做独立对话列表 |

## 当前实现口径（demo）

当前 demo 恢复目标：
- 先用 mock 数据与本地状态完成 V4 主体验
- 不要求打通真实 AI、真实语音、真实后端
- 设置板块当前以“角色选择 + 面板交互”作为基准，不再单独强调独立主题切换入口
- 旧 Phase 2 页面保留文件，但退出主流程

## 不在本版主流程中的能力

以下能力先保留为 Phase 2 / 后续能力，不再进入当前 demo 主链路：
- 会员购买页
- 深度调研页
- 协同筹备页
- 履约页
- 反馈故事卡独立页

它们的文件可保留，但不作为当前用户默认路径。
