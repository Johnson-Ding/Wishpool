# Wishpool V2.1 Home Feed 需求变更设计（A+C）

- 日期：2026-03-21
- 状态：已与用户确认，待进入 implementation planning
- 范围：仅 `HomeScreen`（`demo/client/src/pages/WishpoolDemo.tsx`），不改其他屏幕
- 对应需求：
  - A：Home Feed 按 PRD V2.1 全量更新
  - C：Home Feed 全卡片评论（半屏抽屉 + 语音假转写）

---

## 1. 变更目标与边界

### 1.1 目标（In Scope）

1. Home Feed 支持 V2.1 内容流的交互行为与按钮映射（含 `story/mumble/news/rec/goodnews/poem/quote`）。
2. `story` 支持分支：
   - 单人：`帮Ta实现 →` + `我也想做`
   - 多人（`meta` 含“助力”）：`参与其中 →` + `我也想做`
3. 全 Home Feed 卡片支持评论入口。
4. 评论交互采用半屏抽屉（sheet），支持：
   - 文本输入
   - 语音录入态 UI
   - 假转写回填输入框
5. 保持现有首页滑动、点赞基础体验可用。

### 1.2 非目标（Out of Scope）

1. 不扩展评论到 `FeedbackScreen` 故事卡。
2. 不接入真实语音识别服务。
3. 不拆分文件（仍保持单文件实现）。
4. 不新增状态管理库或第三方依赖。
5. 不改变全局主题变量命名。

### 1.3 必须保持约束

- 遵守项目协作规范：Phase 3、单屏最小改动、主题感知、手机优先。
- 不引入横向滚动，不破坏 375×812 手机壳展示。
- 不使用中文弯引号导致编译错误。

---

## 2. 当前行为基线（Baseline）

基于 `WishpoolDemo.tsx` 当前 `HomeScreen`：

1. 已存在卡片滑动浏览与进度指示。
2. 已有类型分支按钮（部分 V2.1 映射已具备雏形）。
3. 已有点赞逻辑（`likedIds` + 心形动画）。
4. 尚无统一评论抽屉能力。
5. `帮Ta实现/参与其中` 缺少可见的任务半屏闭环。

---

## 3. 目标交互设计

## 3.1 按钮映射规则

| 卡片类型 | 主按钮 | 次按钮 | 通用操作 |
|---|---|---|---|
| `story` 单人 | 帮Ta实现 → | 我也想做 | ❤️ + 评论 |
| `story` 多人（meta 含“助力”） | 参与其中 → | 我也想做 | ❤️ + 评论 |
| `mumble` | 深有同感 | — | ❤️ + 评论 |
| `news` / `rec` | 我也想做 → | — | ❤️ + 评论 |
| `goodnews` / `poem` / `quote` | 真好 | — | ❤️ + 评论 |

## 3.2 行为语义

1. `我也想做`：复用现有发愿入口逻辑（会员流不变）。
2. `帮Ta实现 →`：打开“线上任务”半屏（内容核验/转帖入口）。
3. `参与其中 →`：打开“多人任务详情”半屏（确定参与/先不了）。
4. `深有同感` / `真好`：即时正反馈并切换下一张（保持轻互动感）。
5. `评论`：打开半屏评论抽屉。

---

## 4. 状态模型（HomeScreen 内）

新增状态（建议）：

- `activeCommentBottleId: number | null`
- `commentDraftById: Record<number, string>`
- `commentCountBoostById: Record<number, number>`
- `isRecording: boolean`
- `recordingBottleId: number | null`
- `taskSheetBottleId: number | null`
- `taskSheetMode: "assist" | "collab" | null`
- `toast: { text: string; visible: boolean }`

保留状态：

- `current` / `direction`（卡片切换）
- `likedIds` / `likeAnim`（点赞）

状态约束：

1. 同时仅允许一个评论抽屉。
2. 评论草稿按卡片隔离。
3. 抽屉关闭时，录音态自动结束。

---

## 5. 评论抽屉设计（C）

## 5.1 抽屉结构

1. Header：卡片标签 + 标题摘要（帮助用户确认评论对象）。
2. Body：多行文本输入框。
3. Footer：
   - 语音按钮（录音态动画）
   - 发送按钮

## 5.2 语音假转写（B 档）

1. 点击语音按钮 → 进入录音态 `isRecording=true`。
2. 约 1.5~2 秒后自动结束。
3. 将模拟转写文本回填当前卡片输入框。

## 5.3 发送行为

1. 发送成功（本地）后：
   - 当前卡片评论数本地 +1
   - 关闭抽屉
   - toast 提示“评论已发送”
2. 空文本发送：按钮置灰或拦截，不做提交。

---

## 6. 任务半屏设计（A）

## 6.1 单人故事：帮Ta实现

半屏展示两个入口：

1. 内容核验（AI 提问任务）
2. 帮忙转帖（扩散任务）

点击“接受任务”后：toast 提示 + 关闭半屏（本轮不做深页）。

## 6.2 多人故事：参与其中

半屏展示：

- 发起人、参与人数、地点/时间（若有）、任务说明
- 操作：`确定参与` / `先不了`

本轮行为：本地确认反馈 + 关闭半屏（不跨屏跳转，控制范围）。

---

## 7. 实现边界与最小重构

允许在 `HomeScreen` 内做轻量重构，建议新增内部函数：

1. `getCardActionConfig(card)`：统一映射文案和行为类型。
2. `renderCardActions(card)`：渲染按钮区（主/次/点赞/评论）。
3. `renderCommentSheet()`：渲染评论抽屉。
4. `renderTaskSheet()`：渲染任务抽屉。

不做：

- 跨文件组件抽离
- 全局通用评论系统抽象

---

## 8. 验收标准（可测试）

1. 所有 Home Feed 卡片类型文案映射正确。
2. `story` 助力判定正确触发“参与其中”。
3. 所有 Home 卡片均可打开评论抽屉。
4. 评论语音按钮可产生录音态并回填模拟文本。
5. 评论发送后，计数 +1 且出现反馈提示。
6. `帮Ta实现/参与其中` 都可打开对应半屏。
7. 无编译错误、无白屏、无横向滚动。

---

## 9. 风险与回滚

### 9.1 风险

1. HomeScreen 交互状态增加，可能出现状态冲突（抽屉/滑动/点赞并发）。
2. 按钮映射分支较多，容易出现类型遗漏。
3. 半屏层级可能影响手势与点击穿透。

### 9.2 回滚

- 代码为单文件局部改动，可通过单次 git revert 回滚。
- 若仅交互异常，可先回滚评论/任务抽屉入口，保留现有浏览与点赞主链路。
